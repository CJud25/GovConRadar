# Data dictionary (v2.0.0 additions)

This extends the base per-column reference in [`data_dictionary.csv`](data_dictionary.csv) (generated by
`src/export/documentation.py`). Below are the **v2.0.0** derived columns and the KPI overhaul. Schema
changes are **additive**: every legacy column stays populated for Power BI compatibility.

## `fact_recompete_candidates` — new derived columns

| Column | Type | Meaning |
|---|---|---|
| `candidate_status` | str | `active` (days ≥ 0) / `expired_grace` (−90..−1) / `expired_stale` (< −90). Missing runway → `expired_stale` (conservative quarantine). |
| `title_display` | str | Display-safe title. IGF codes stripped; a garbled/raw-record title collapses to `[Untitled award — see source record]`. **The raw `contract_title` is never shown in the UI.** |
| `flag_garbled_title` | bool | Title is a raw pipe/bang-delimited FPDS record dump or unusable junk. |
| `flag_code_prefix` | bool | Title carries a raw `IGF::CT::` / `IGF::OT::` code prefix. |
| `flag_short_title` | bool | Title < 10 chars after trim (real but low-information). |
| `flag_stale_expiration` | bool | Expired > 90 days ago → Data Gap. |
| `flag_missing_end_date` | bool | No usable expiration date → Data Gap. |

**Changed semantics (same column names):**
- `days_until_expiration` — now recomputed from `selected_expiration_date` vs. the snapshot date at
  bake, and vs. **today** at app load. (Was baked once and drifted.)
- `expiration_bucket` (+ `expiration_bucket_sort`) — new set: `Expired — verify` (gray quarantine,
  sorts first), `0-6`, `6-12`, `12-18`, `18-24`, `24+ Months`. No expired row appears in a forward bucket.
- `priority_tier` — a `Data Gap` value is now assigned by **override** (stale / missing-end-date /
  garbled), never by score. `pursuit_score` still ranges 0–100; the Data Gap tier ignores it.

## `dashboard_kpi_summary` — legacy kept, v2 added

Legacy columns (`total_estimated_pipeline_value`, `recompete_candidate_count`, `tier_1_count`,
`expiring_within_12_months_count`, `top_agency_by_pipeline_value`, `top_incumbent_by_expiring_value`,
`average_pursuit_score`, `average_data_quality_score`) remain — **but** `average_data_quality_score` is
now computed under the fixed logic (no longer a fake 100.0), and `tier_1_count` excludes stale records
(they are Data Gap). Added:

| Column | Meaning |
|---|---|
| `snapshot_date` | Data as-of date (max `pull_timestamp_utc`), the runtime source of truth (not file mtime). |
| `scorer_version` | Must equal `rescore.SCORER_VERSION` (validator-enforced). |
| `active_candidate_count`, `active_pipeline_value`, `active_tier_1_count` | The honest headline (active only). |
| `expired_grace_count`, `expired_stale_count`, `expired_pending_value` | The quarantine split. |
| `top_dod_component_by_active_value` | The real discriminator (agency is a DoD-wide constant). |
| `top_incumbent_by_active_value` | Top incumbent by active pipeline value. |
| `vehicle_count`, `task_order_count` | Distinct IDVs vs. task orders among active candidates. |
| `median_active_runway_days` | Median runway of active candidates. |
| `garbled_title_count`, `code_prefix_title_count`, `stale_expiration_count` | Flag totals. |

## `dim_agency` / `dim_vendor` — additive rollups

All legacy columns retain their original all-rows semantics (documented as legacy). Added:
`dim_agency.active_candidate_count`, `active_pipeline_value`, `active_tier_1_count`;
`dim_vendor.active_recompete_candidate_count`, `active_value`.

## `data_quality_report` — new metrics

Existing row-count/missing-value/limitation rows are kept. Added flag metrics (`garbled_title_count`,
`code_prefix_title_count`, `short_title_count`, `stale_expiration_count`, `missing_end_date_flag_count`)
and stale-age bands (`expired_grace_count_0_90d`, `expired_stale_count_90d_1y`, `_1y_5y`, `_over_5y`).

## Factual (read-only) vs. derived

**Facts** (never mutated by the rebake): identifiers, `contract_title`, all dates, dollars, agency/
subagency/incumbent, NAICS/PSC, competition & set-aside codes, `source_url`, `data_quality_notes`, all
`ptw_*`. **Derived** (regenerated): everything in the tables above plus surrogate `*_key` columns,
`capture_phase`, and all rollups/KPIs.
